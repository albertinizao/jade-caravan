# Licencia del proyecto Grimorio Pathfinder

> Documento de licencia para el repositorio.
>
> Este documento no constituye asesoramiento jurídico profesional. Su objetivo es dejar explícito
> cómo debe tratar este repositorio el código propio, la documentación, los datos de conjuros,
> las traducciones, los overrides y el contenido usado bajo la Open Game License v1.0a.

## 1. Naturaleza del proyecto

**Grimorio Pathfinder** es una aplicación local y offline para consulta, búsqueda y edición de
conjuros compatibles con **Pathfinder Roleplaying Game 1ª edición**.

Este proyecto no es un producto oficial de Paizo Inc., Wizards of the Coast, Inc. ni de ningún
otro titular relacionado con Pathfinder, Dungeons & Dragons, d20 System u Open Game License.

**Pathfinder** es una marca de Paizo Inc. Cualquier referencia a Pathfinder se realiza únicamente
para describir el sistema de juego con el que se pretende usar esta herramienta.

Este proyecto no debe usar logos oficiales, trade dress, ilustraciones oficiales, mapas oficiales,
personajes de ambientación, nombres propios protegidos, textos narrativos cerrados ni otros
elementos designados como Product Identity, salvo autorización expresa del titular correspondiente.

## 2. Separación entre código, documentación y contenido de juego

Este repositorio distingue entre:

1. **Código fuente original de la aplicación**.
2. **Documentación técnica original del proyecto**.
3. **Datos generados o importados de conjuros**.
4. **Correcciones manuales, traducciones y notas personales**.
5. **Open Game Content usado bajo la Open Game License v1.0a**.

Cada tipo de contenido puede estar sujeto a condiciones distintas.

## 5. Contenido de juego y Open Game Content

Determinados datos de juego usados por este proyecto pueden proceder de obras publicadas bajo la
**Open Game License Version 1.0a**.

Cuando este repositorio copie, traduzca, modifique, transforme, derive, compile o redistribuya
Open Game Content, ese contenido se usa exclusivamente bajo los términos de la Open Game License
v1.0a.

A efectos de este proyecto, se considerará Open Game Content, salvo que esté excluido como Product
Identity:

- reglas mecánicas abiertas;
- nombres de conjuros cuando formen parte de Open Game Content;
- niveles de conjuro;
- escuelas, subescuelas y descriptores;
- componentes;
- tiempos de lanzamiento;
- alcance, objetivo, efecto, área y duración;
- tiradas de salvación;
- resistencia a conjuros;
- descripciones mecánicas de conjuros procedentes de fuentes OGL;
- traducciones al español de Open Game Content;
- correcciones manuales que modifiquen o traduzcan Open Game Content;
- datos estructurados derivados de Open Game Content.

El contenido anterior no queda relicenciado bajo la licencia del código fuente. Permanece sujeto a la
Open Game License v1.0a.

## 6. Product Identity y contenido excluido

Este proyecto no declara como Open Game Content ningún elemento que sea Product Identity de Paizo
Inc., Wizards of the Coast, Inc. u otros titulares.

Quedan excluidos de cualquier concesión de licencia realizada por este repositorio:

- marcas registradas;
- logos;
- trade dress;
- arte e ilustraciones;
- mapas;
- maquetación visual característica de libros oficiales;
- personajes propios de ambientaciones oficiales;

- nombres de lugares, organizaciones o tramas de ambientaciones oficiales;
- textos narrativos no mecánicos;
- aventuras, tramas, storylines y diálogos;

La presencia accidental de alguno de estos elementos en el repositorio no implica que se conceda
licencia sobre él. Si se detecta contenido cerrado o Product Identity no autorizado, deberá eliminarse
o sustituirse.

## 7. Traducciones y correcciones manuales

Las traducciones españolas de Open Game Content se consideran material derivado del Open Game Content
original y se distribuyen bajo la Open Game License v1.0a cuando se distribuyan fuera del uso privado
del titular.

Las correcciones manuales sobre campos de conjuros procedentes de Open Game Content también se
consideran derivadas de ese Open Game Content cuando modifiquen, reescriban o traduzcan reglas o texto
mecánico.

Las notas personales del usuario se tratarán como contenido privado del titular del repositorio, salvo
que:

- copien o reproduzcan Open Game Content;
- modifiquen texto mecánico OGL;
- se incluyan deliberadamente en una exportación distribuida como Open Game Content;
- el titular del repositorio las marque expresamente como Open Game Content.

## 8. Datos generados, importados y overrides

Los archivos de datos del proyecto pueden contener una mezcla de:

- datos originales del proyecto;
- Open Game Content;
- traducciones de Open Game Content;
- correcciones manuales;
- notas personales;
- metadatos técnicos.

La aplicación debe mantener separada, en la medida de lo posible, la procedencia y naturaleza de cada
tipo de dato.

En particular:

```text
data/generated/
```

puede contener datos generados a partir de fuentes OGL o datos propios del proyecto.

```text
data/overrides/
```

puede contener correcciones manuales, estados de revisión y notas personales.

```text
data/raw/
```

puede contener fuentes originales usadas para trazabilidad, siempre que su inclusión en el repositorio
esté permitida por la licencia aplicable.

No debe añadirse al repositorio ningún archivo fuente cuya licencia no permita copia, modificación o
almacenamiento en el repositorio.

## 9. Obligaciones al distribuir contenido OGL

Si este proyecto, total o parcialmente, se distribuye con Open Game Content, deberá cumplirse como
mínimo lo siguiente:

1. Incluir una copia completa y literal de la **Open Game License Version 1.0a**.
2. Identificar claramente qué partes del repositorio o de la distribución son Open Game Content.
3. Identificar claramente qué partes son Product Identity o contenido no abierto.
4. Mantener la sección de copyright notice exigida por la Open Game License.
5. Añadir al copyright notice todas las obras OGL de las que se haya copiado, traducido, modificado o
   derivado contenido.
6. No usar marcas, logos o Product Identity para indicar compatibilidad salvo que exista una licencia
   independiente que lo permita.
7. No imponer condiciones adicionales sobre el Open Game Content que contradigan la Open Game License
   v1.0a.

## 10. Declaración de Open Game Content de este proyecto

Salvo indicación más específica en un archivo concreto, este proyecto designa como Open Game Content
únicamente las partes de los archivos de datos que deriven de contenido previamente publicado como Open
Game Content.

Esto incluye, cuando existan:

```text
data/generated/spells-es.generated.json
data/exports/
```

y las partes de:

```text
data/overrides/spells-es.overrides.json
```

que modifiquen, corrijan o traduzcan texto mecánico OGL.

No se designan como Open Game Content:

- código fuente de backend;
- código fuente de frontend;
- scripts de importación;
- documentación técnica original;
- estructura visual de la interfaz;
- nombre del proyecto;
- logotipos propios;
- notas personales que no deriven de OGL;
- configuraciones locales;
- archivos privados;
- contenido de desarrollo interno.

## 11. Declaración de Product Identity del proyecto

En la medida permitida por la Open Game License, se designan como Product Identity propia del proyecto:

- el nombre interno del proyecto, excluyendo cualquier marca de terceros;
- logotipos originales del proyecto, si existen;
- diseño visual original de la aplicación;
- documentación técnica original no derivada de Open Game Content;
- código fuente original;
- estructura de paquetes, arquitectura y nombres técnicos propios;
- textos explicativos originales no derivados de reglas OGL.

Esta declaración no reclama propiedad sobre marcas de terceros, incluyendo Pathfinder, Paizo, Dungeons
& Dragons, d20 System, Wizards of the Coast ni otras marcas ajenas.


## 13. Archivo OGL complementario

Este repositorio debe incluir el archivo:

```text
NOTICE-OGL.md
```

Ese archivo debe contener:

- una copia completa y literal de la Open Game License Version 1.0a;
- la identificación de Open Game Content y Product Identity aplicable al repositorio;
- la Section 15 / Copyright Notice correspondiente a las fuentes OGL realmente usadas.

Si el repositorio aún no incluye datos OGL reales, `NOTICE-OGL.md` puede mantenerse como plantilla de
preparación, pero no debe distribuirse contenido OGL público con la Section 15 incompleta.

## 15. Regla práctica para contribuciones futuras

Toda contribución al repositorio deberá respetar estas reglas:

1. No copiar contenido cerrado de libros, PDFs, webs o productos oficiales.
2. No copiar Product Identity salvo permiso expreso.
3. No añadir logos ni arte oficial.
4. No añadir textos de aventura, ambientación, personajes o deidades oficiales.
5. No añadir contenido de una fuente si no está claro que sea Open Game Content o contenido propio.
6. Marcar la procedencia de cualquier dato OGL importado.
7. Añadir la entrada correspondiente al Section 15 si se incorpora una nueva fuente OGL.
8. Mantener separadas las licencias de código, documentación y contenido de juego.

## 16. Aviso final

Este documento pretende dejar explícito el tratamiento de licencias dentro del repositorio, pero no
constituye asesoramiento jurídico profesional.

Si el proyecto se distribuye públicamente, se comercializa o se publica con una cantidad significativa
de contenido derivado de Pathfinder u otras obras OGL, conviene revisar la licencia final con una
persona cualificada en propiedad intelectual y licencias de software/juegos.
